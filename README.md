## 為什麼是 Claude

試過各家之後留在 [Claude](https://claude.ai)。沒什麼信仰上的理由，純粹是用起來最順。它讀 code 的能力讓我省了很多時間，回應的品質也比較穩定。

現在的開發流程大概是這樣：

- 技術選型丟給 Claude 分析 trade-off，我做決定
- 架構用自然語言描述，它出骨架，我審核
- Debug 直接貼 stack trace，省下來的時間拿去喝咖啡
- 與 AI 溝通這件事本身，變成我花最多心思的地方
